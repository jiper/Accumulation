应用场景：
本地工程和云端工程保持同步

本质：
通过SSH服务进行授权，实现本地和云端的加密通信

原理：
SSH协议有一组公钥和私钥，本地和云端通过这一对钥匙进行身份匹配和确认
公钥和私钥是唯一配对的
公钥是钥匙，私钥是锁

方法：
1.本地通过GitExtension生成一对公钥和私钥。（造出了一对钥匙和锁）
2.把公钥提供给云端的GitHub账号（把钥匙给对方）
3.把云端的项目克隆到本地，一定记得配置对应的私钥（告诉云端是开哪一把锁）
4.克隆成功后就实现了本地和云端的同步，本地可以pull和push内容到云端上去了
5.GitHub账号可以同时保存多个SSH公钥，一旦删除，则对应的本地工程无法继续同步到云端了
